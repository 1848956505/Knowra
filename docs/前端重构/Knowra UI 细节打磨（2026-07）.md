# Knowra UI 细节打磨（2026-07）

> 本文档用于指导 Knowra 新 UI 的进一步打磨。
> 本文档不是最终视觉稿，而是当前阶段已经确定的设计思想、信息架构以及后续优化方向。
>
> **重要：**
> AI（Codex）可以继续优化视觉细节，但不要破坏本文已经确定的信息架构与交互原则。

---

# 一、整体设计目标

Knowra 的定位不是：

- IDE
- 文档编辑器
- Todo 软件

而是：

> **Knowledge Operating System（知识操作系统）**

因此整个 UI 需要满足：

- 长时间阅读
- 长时间整理知识
- 多模块切换
- 强沉浸感
- 高扩展性

整体风格目标：

> **现代、克制、专业、具有品牌辨识度。**

关键词：

- Swiss
- Editorial
- Archive
- Minimal
- Modern

---

# 二、整体布局（保持三栏）

整个应用保持：

```
┌──────────┬──────────────────────┬─────────────┐
│ 左侧导航  │ 编辑区 / 主内容区       │ 右侧信息栏    │
└──────────┴──────────────────────┴─────────────┘
```

三栏均独立滚动。

不要让浏览器整体滚动。

---

# 三、左侧导航设计

## 已确定

取消左上品牌展示。

不要：

```
Knowra Logo
```

保留：

```
资料库
LIBRARY
```

或

```
知识库
KNOWLEDGE
```

作为模块 Header。

原因：

Logo 在进入应用以后价值很低。

应把空间留给目录。

---

## 模块 Header

样式保持不变，但是将原本点击logo区域跳转到模块首页的功能迁移到模块Header上，即点击模块Header区域，会跳转到模块首页

---

# 四、右侧信息栏顶部

取消：

大面积蓝色按钮。

例如：

```
打开资料
```

不要再作为 Hero Button。

改成：

```
当前资料名称

MARKDOWN DOCUMENT

打开 →
```

**同时这部分区域要和左侧模块Header大小一致，样式相似。这样左侧和右侧两栏视觉上能够更统一**

---

# 五、中间区域

中间 Header 不再直接重复：

```
资料库
```

推荐：

```
资料索引

LIBRARY INDEX
```

避免：

左栏：

资料库

中间：

资料库

造成重复。

---

# 六、编号设计

经过讨论：

编号虽然信息价值较低。

但是：

它形成了品牌语言。

因此：

Header 保留：

```
01

资料库
```

同时编号样式不要太粗，
