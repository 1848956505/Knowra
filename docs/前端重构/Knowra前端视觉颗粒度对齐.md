# Knowra 前端视觉颗粒度对齐

> 状态：持续补充。本文件记录已经确认的视觉与交互共识；尚未确认的内容不作为重构实现依据。

## 1. 使用方式

- 视觉基线：`工作台（首页）.png`、`资料库.png` 与 `原始稿/knowra-swiss-grid/`。
- 每次只对齐一个区域；确认后补入“已确认规则”，再进入工程拆分与实现。
- 原型只验证视觉、空间和交互，不代表后端接口、数据模型或最终文案已经确定。
- 对齐层级依次为：页面骨架 → 区域结构 → 组件形态 → 状态与交互 → 响应式与可访问性。

## 2. 当前 UI 风格基线

### 2.1 设计语言

- 风格：瑞士编辑网格（Swiss Editorial Grid）。
- 气质：理性、锐利、专业、编辑型、高信息密度，避免普通 SaaS 卡片感。
- 主要秩序来源：严格网格、清晰分隔线、编号、字号层级和留白，而不是圆角、阴影或装饰。

### 2.2 色彩

| 角色 | 当前基线 | 用途 |
| --- | --- | --- |
| 页面底色 | 象牙白 / 冷灰白，原始稿约 `#f5f3ec` | 全局背景与连续工作面 |
| 主文字 | 近黑，原始稿约 `#161616` | 标题、正文、结构信息 |
| 主强调 | 高饱和钴蓝，原始稿约 `#1646d8` | 当前项、主操作、编号、链接 |
| 状态强调 | 荧光黄绿，原始稿约 `#e9ff4f` | 少量状态点，不作大面积装饰 |
| 分隔线 | 中性灰，原始稿约 `#9c9b94` | 网格边界、栏位和列表规则线 |

规则：钴蓝必须表达“当前、可操作或关键”，荧光黄只承担状态提示；不能给同级模块随意分配不同颜色。

### 2.3 字体与层级

- 中文使用清晰的无衬线黑体；英文编号优先使用窄体无衬线。
- 一级页标题超大、重字重、紧字距；模块编号比模块名更醒目。
- 正文与辅助信息保持克制，不用大量粗体争夺层级。
- 中文功能名为第一识别层，英文名称可作小号辅助层。

### 2.4 几何与组件

- 以直角或极小圆角为主。
- 结构分隔以 `1px` 规则线为主，选中项可增加一条钴蓝竖线。
- 按钮、标签、输入框不依赖阴影；主按钮使用实心钴蓝。
- 列表和编辑区优先连续铺陈，不使用重复悬浮卡片包裹。

### 2.5 页面骨架

桌面端当前基线由三类垂直区域构成：

1. 左侧一级导航 / 模块上下文；
2. 中央主工作区；
3. 按页面需要出现的右侧详情、边注或上下文面板。

顶部标签栏、底部状态栏属于编辑场景，不要求在主工作台常驻。

## 3. 已确认区域：左侧导航栏

### 3.1 职责

- 一级模块固定为 `01～05`，负责切换产品模块。
- 当前模块的二级目录在左栏的固定目录位展示。
- 收藏、最近、归档属于模块内部信息架构，不作为全局一级入口。
- 底部保留设置；Logo / 品牌标识是明确的工作台入口。

### 3.2 固定模块列 + 固定目录位

- `01～05` 永不换位，保留用户的空间记忆。
- 默认状态下模块项以大号编号和完整名称展示。
- 任一模块激活后，五个一级模块统一压缩为紧凑行，给目录区腾出稳定空间。
- 目录区始终从同一个坐标开始，不跟随被点击模块的位置。
- 目录区顶部重复当前模块的编号和名称，用钴蓝与规则线建立上下文。
- 再次点击当前模块返回主工作台；Logo / 工作台入口也可显式返回。

### 3.3 状态模型

```text
activeModuleId     当前一级模块；为空时为主工作台
activeSecondaryId  当前二级目录；仅在模块内有效
contextPanelOpen   固定目录位是否展开
focusMode          是否进入编辑专注态
```

### 3.4 四种需要覆盖的场景

| 场景 | 左栏 | 中央区域 |
| --- | --- | --- |
| 主工作台 | 五个完整模块项，无目录 | 全局搜索、最近访问和跨模块索引 |
| 模块展开 | 五个紧凑模块项，固定目录位展开 | 当前模块概览 |
| 二级目录选中 | 同上，目录项有明确选中态 | 对应列表或功能页 |
| 编辑专注 | 模块列降噪，目录位收起或变窄 | 编辑器成为主视觉 |

### 3.5 暂未锁定

- 五个模块的最终中文主名称与主题副标题。
- 左栏最终宽度、紧凑行高度和目录最大层级。
- 窄屏下使用抽屉、底栏还是独立导航页。
- 编辑专注态是否保留完整模块名称。
