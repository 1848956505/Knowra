# Knowra UI 重构迁移路线审查

本目录用于固定 2026-07-15 阶段的 UI 重构路线结论、视觉证据与下一阶段执行边界。

## 当前结论

如果在以下两种方案中二选一：

1. 直接把新 UI 套入正式项目，再边开发边打磨；
2. 先打磨测试 demo，再迁移到正式项目；

应选择第 2 种，但不把测试 demo 独立打磨成脱离真实功能的最终成品。

推荐路线是：

> 先把 demo 收敛成可执行的“设计契约”，再以“资料库目录 → 打开资料 → 编辑与保存”为第一条纵向切片迁入正式项目；剩余细节在真实数据和真实编辑器中继续打磨。

- [`02-下一阶段执行方案.md`](./02-下一阶段执行方案.md)：下一阶段的任务顺序、验收门槛和暂不处理范围。
- [`03-颗粒度对齐清单.md`](./03-颗粒度对齐清单.md)：进入设计打磨前需要确认的少量产品决策。
- [`assets/`](./assets/)：本轮审查使用的桌面端页面截图。

## 使用原则

- 《Knowra 左侧导航栏设计说明》继续作为左侧栏当前共识。
- 不再使用已经过时的《Knowra 前端视觉颗粒度对齐》指导后续实现。
- demo 用于验证结构、比例、状态和视觉规则，不作为正式项目的技术实现模板。
- 正式项目继续保留 vanilla JS、Milkdown、现有 API 与状态管理结构，不因 demo 使用 React 而顺带迁移技术栈。
