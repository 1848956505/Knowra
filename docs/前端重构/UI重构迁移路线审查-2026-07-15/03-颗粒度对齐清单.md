# Knowra UI 重构颗粒度对齐清单

以下问题会直接改变下一阶段工作范围。每项均给出推荐默认值；如果没有异议，可以直接按推荐值执行。

## 1. 第一阶段覆盖范围

推荐默认值：**只完整打磨 01 资料库及其编辑器路径，02～05 只保留统一模块入口和轻量占位目录。**

原因：01 资料库已经具备真实目录、资料和编辑器能力，最适合验证新壳层是否能承载正式功能。过早设计其他模块会放大未确定信息。

需要确认：第一阶段是否需要同时设计 02 知识库的完整页面？

## 2. 响应式范围

推荐默认值：**本阶段桌面优先，锁定 `1440 × 1024` 与 `1280 × 800`；移动端仅保证不崩坏，不做最终体验。**

原因：当前参考稿和左侧栏共识都是桌面工作台模型。过早处理移动端会迫使导航结构再次变化。

需要确认：移动端是否属于本轮 UI 重构的硬性验收范围？

## 3. 正式项目功能保留原则

推荐默认值：**所有现有功能最终必须保留；迁移过程中允许按纵向切片逐步恢复，但不允许通过删除功能来适配新 UI。**

原因：正式项目现有的编辑器、附件、标签、目录命令和快捷键是产品能力，不应被视觉重构隐式降级。

需要确认：是否存在确定要随 UI 重构一起下线的旧功能？

## 默认执行口径

如果以上三项均采用推荐默认值，则下一轮直接进入：

> 仅打磨测试 demo 的 01 资料库桌面端四状态，并以完整保留正式项目前端能力为迁移约束。
