# Knowra 左侧导航栏设计说明

### 0.整体UI设计：瑞士编辑网格

大胆、清晰、专业，特色最鲜明。

- 背景：纯净象牙白或冷灰白
- 主色：高饱和钴蓝，可少量加入荧光黄
- 布局：严格网格、大字号章节编号、明显的纵横分隔
- 导航更像杂志目录，知识点像索引卡或研究档案
- 边角偏小，不依赖常见的“大圆角悬浮卡片”
- 信息密度最高，但借助字号、线条和色彩建立秩序
- 参考气质：瑞士国际主义平面设计、现代研究工具、编辑型网站
- 适合：研究、论文、专业知识管理，以及希望避开“普通 SaaS 模板感”的产品

视觉关键词：`理性 / 锐利 / 编辑网格 / 高辨识度`

## 1. 设计定位

Knowra 左侧导航栏不是传统意义上的普通菜单栏，而是整个应用的一级功能切换器。

左侧导航栏主要承担两个职责：

1. 切换 Knowra 的一级功能模块；
2. 在当前功能被激活后，展开该功能对应的二级目录。

左侧导航栏不承担收藏、最近、归档等内容筛选职责，这些内容应放入主工作台或对应功能页面中处理。

## 2. 整体结构

左侧导航栏由三部分组成。

### 2.1 顶部

放置Logo，如工作台图片所示

### 2.2 中部

放置 01～05 五个一级功能按钮。

当前规划如下：

```text
01 银盐档案馆
02 知忆穹廊
03 侘寂浮岛
04 时契罗经仪
05 观测所
```

其中：

- 01 是资料库：管理原始资料、笔记、文件夹、附件等；
- 02 是知识库：管理知识点、标签、知识关系等；
- 03 是试题库：管理题库、错题、练习、考试等；
- 04 是任务与待办：管理任务、计划、今日待办等；
- 05：是AI助手

01～05 是 Knowra 的一级功能模块，不是知识领域，也不是普通文件分类。

### 2.3 底部

仅保留设置入口。

建议形式：

```text
⚙ 设置
```

底部不保留以下入口：

```text
收藏
最近
归档
```

## 3. 默认状态

当没有任何一级功能被选中时，左侧导航栏只展示：

```text
Logo

01 资料库
02 知识库
03 试题
04 任务与待办
05 待定

设置
```

此时中央区域进入主工作台。

主工作台主要承担：

- 全局搜索；
- 全局筛选；
- 全局索引；
- 最近访问；
- 跨模块内容聚合。

主工作台不属于 01～05 中的任何一个功能模块。

## 4. 模块展开交互

左侧导航采用手风琴式交互。

核心规则：

> 同一时间最多只允许一个一级功能模块展开。

例如点击 01 资料库后：

```text
01 资料库
   全部资料
   最近打开
   收藏
   文件夹
   ...

02 知识库
03 试题
04 任务与待办
05 待定
```

点击 02 知识库后：

- 01 自动收起；
- 02 自动展开；
- 中央工作区切换到知识库页面。

---

## 5. 当前模块置顶规则

为了避免点击靠下的模块后，展开目录位置过低，当前选中的模块需要自动移动到顶部。

例如默认顺序为：

```text
01
02
03
04
05
```

点击 05 后，显示为：

```text
05
   二级目录
   二级目录
   二级目录

01
02
03
04
```

因此，无论用户点击 01、03 还是 05：

- 当前模块始终位于顶部；
- 二级目录始终从相同位置展开；
- 用户能够形成稳定的视觉预期和操作习惯。

---

## 6. 其他模块的变化

当某个模块被选中并展开时，其他模块需要进入压缩状态。

### 6.1 当前模块

当前模块应具备：

- 位于顶部；
- 保持完整标题；
- 显示模块编号；
- 显示二级目录；
- 使用明确但克制的选中状态；
- 保留足够空间展示目录。

示例：

```text
01
资料库

全部资料
最近打开
收藏
文件夹
```

### 6.2 非当前模块

其他模块应：

- 保持可点击；
- 不变灰到像“不可用”；
- 缩小高度；
- 隐藏次要描述；
- 紧凑排列在当前模块下方。

示例：

```text
02 知识库
03 试题
04 任务
05 待定
```

非当前模块只是处于未展开状态，不是禁用状态。

---

## 7. 再次点击当前模块

当用户再次点击已经展开的模块时：

- 当前模块收起；
- 所有模块恢复默认顺序；
- 当前没有任何功能模块被选中；
- 中央区域返回主工作台。

例如：

```text
点击 01
→ 01 展开并置顶
→ 再次点击 01
→ 01 收起
→ 返回主工作台
```

---

## 8. 页面切换规则

一级模块的点击不仅改变左侧导航状态，也会切换中央工作区。

### 点击一级模块

例如点击 01 资料库：

- 01 移动到顶部；
- 资料库目录展开；
- 中央区域进入资料库首页。

### 点击二级目录

例如点击“Transformer”文件夹：

- 01 保持展开；
- 当前二级目录高亮；
- 中央区域显示对应内容。

### 再次点击一级模块

- 一级模块收起；
- 返回主工作台。

---

## 9. 状态区分

左侧导航中需要区分三种状态。

### 9.1 默认状态

- 没有任何模块展开；
- 所有模块保持普通状态；
- 中央区域显示主工作台。

### 9.2 模块展开状态

- 一个一级模块置顶；
- 该模块展开二级目录；
- 其他模块压缩排列。

### 9.3 二级目录选中状态

- 一级模块保持选中；
- 某个二级目录高亮；
- 中央区域显示具体页面。

一级模块选中状态与二级目录选中状态不能混淆。

---

## 10. 动画建议

模块切换建议加入轻量动画，但不应过度强调。

推荐动画内容：

- 当前模块移动到顶部；
- 其他模块重新排序；
- 当前模块高度展开；
- 二级目录淡入；
- 非当前模块高度压缩。

建议动画时长：

```text
180ms ～ 280ms
```

动画应体现稳定、克制和高效，不使用弹跳、过度缩放或夸张缓动。

---

## 11. 实现建议

可以使用纵向 Flex 布局。

示意：

```css
.module-list {
  display: flex;
  flex-direction: column;
}

.module.active {
  order: -1;
}
```

核心状态：

```js
activeModuleId
```

渲染逻辑：

```text
有 activeModuleId：
当前模块置顶并展开；
其他模块压缩排列。

没有 activeModuleId：
恢复 01～05 默认顺序；
进入主工作台。
```

推荐通过状态驱动渲染，不要直接依赖大量 DOM 手动移动。

## 13. 最终原则

Knowra 左侧导航栏应遵循以下原则：

1. 只承载一级功能模块和当前模块目录；
2. 01～05 是产品模块，不是知识分类；
3. 同时只展开一个模块；
4. 当前模块置顶；
5. 其他模块压缩但保持可用；
6. 再次点击当前模块返回主工作台；
7. 收藏、最近、归档不作为固定一级入口；
8. 顶部只放 Logo；
9. 底部只放设置；
10. 左侧导航应保持稳定、克制、清晰，并成为 Knowra 的核心视觉识别之一。
