# 工程变更日志

本文档是 **Study Accelerator** 项目的**唯一**变更日志，主要读者是 **AI Agent**。

本文档同时承担「产品变更日志」与「工程变更日志」两种职责：
- 「产品变更」放在 `Added` / `Changed` / `Fixed` / `Removed` / `Deprecated` 章节
- 「纯工程变更」放在 `Refactor` 章节
- 「工具/构建/CI 变更」放在 `Chore` 章节

> 📌 **维护原则（给 Agent 看）**
> 1. 每次完成结构性变更（新增/拆分/迁移文件、依赖升级、构建脚本、API 行为变化）必须**在本文档对应版本下加一条**
> 2. 每条记录末尾用**括号标注平台**（见下方规则）
> 3. 旧版本不要修改，只在 [Unreleased] 或新版本下追加
> 4. 写完后 grep 验证段落结构没坏

## 平台/系统标注规则

每条记录末尾用括号标注涉及的平台，便于 Agent 快速判断是否需要考虑平台差异。

- **不标 = 默认全平台**（macOS / Windows / Linux/Ubuntu / Web）都行
- 常见标签：

  | 标签 | 含义 |
  | --- | --- |
  | `macOS` | 仅 macOS 客户端 |
  | `Windows` | 仅 Windows 客户端 |
  | `Linux/Ubuntu` | 仅 Linux/Ubuntu 客户端或服务器 |
  | `Server` | 仅服务端（部署在 Linux/Ubuntu 上的 Node 进程） |
  | `Web` | 仅浏览器端运行 |
  | `多端` | 涉及 2 个及以上平台 |

- 示例：
  - `（Server）` —— 仅服务端 Linux/Ubuntu
  - `（macOS）` —— 仅 macOS 客户端
  - `（多端）` —— 涉及 2 个及以上平台
  - `（macOS + Server）` —— 同时涉及两端

## [Unreleased]

> 下一版本（`2.0.7` 或 `2.1.0`）待合并的变更。

### Fixed
- 修复 highlight（Ctrl+Shift+H / 工具栏）报 `undefined is not an object (evaluating 'slice2.id')` 错误：根因是 `editor-factory.js` 漏了 `.use(toggleHighlightCommand)`，导致 `toggleHighlightCommand.key` 永远是 `undefined`，按快捷键时 `callCommand(undefined)` 触发 `ctx.get(undefined)` 抛错（多端）

### Added
- 粘贴 AI 工具（ChatGPT / Claude）回复时自动清洗 inline `style` 属性：新增 `apps/web/lib/editor/markdown-paste.js` 的 `stripPastedInlineStyles(html)` 函数 + `markdown-paste-plugin.js` 的 `transformPastedHTML` ProseMirror 钩子，解决淡蓝紫关键词背景污染笔记的问题（多端）
- 在 `docs/开发规范.md`「## ⛔ 核心铁律」新增第 5 条「改前同步原则」：开始改代码前必须 `git fetch origin && git status` 确认本地是远端最新状态，避免在独立 commit 链上工作。AI 自检清单对应新增第 11 条（多端）

### Tests
- `apps/web/test/markdown-paste.test.js` 新增 5 个断言覆盖 `stripPastedInlineStyles`：非 string 透传、空串 no-op、null 透传、无 DOMParser 透传、有 DOMParser 删除 style 但保留文本。web tests 113/113 通过

## [2.0.0 ~ 2.0.5] - 2026-07-04（项目初始化里程碑）

> v2.0.0 ~ v2.0.5 是 [`docs/V2.0开发计划.md`](../V2.0开发计划.md) 中的开发计划路线图，作为里程碑一起记录在此。
> 本仓库 git 起点是 `43cae67 feat: 初始化 Knowra 项目并修复跨平台编辑器兼容性问题`。
> 之前的 v1.x 历史规划见 [`docs/已归档/`](../已归档/)。

### Added

#### 项目骨架
- 项目初始化（commit 43cae67）：本地优先知识库工作台（多端）
- 后端：plain Node.js HTTP API + DDD 风格分层（`apps/api/src/{domain,application,infrastructure,http}`）（Server）
- 前端：vanilla JS SPA + 模块化拆分（`apps/web/src/{app,controllers,services,server,client.js}` + `apps/web/lib/*`）（Web）
- 编辑器：Milkdown 集成（`apps/web/lib/editor/milkdown-entry.js` + bundle 产物）（Web）
- 数据持久化：本地 JSON 文件 `storage/data/knowledge-base.json`（Server）
- 自动化测试：API + Web 都用 Node.js 自定义 runner（无 Jest/Mocha）（多端）
- 部署：阿里云 ECS + Nginx + PM2（Server）
- 客户端覆盖：macOS / Windows / Web 三端均验证可运行（多端）

#### v2.0.0 数据底座
- 知识点能力：新增实体 `KnowledgePoint`、`KnowledgePointSource`、`TagGroup`、`KnowledgePointTag`、`NoteKnowledgePoint`（Server）

#### v2.0.1 标签体系
