# 工程变更日志

本文档是 **Study Accelerator** 项目的**唯一**变更日志，主要读者是 **AI Agent**。

本文档同时承担「产品变更日志」与「工程变更日志」两种职责：
- 「产品变更」放在 `Added` / `Changed` / `Fixed` / `Removed` / `Deprecated` 章节
- 「纯工程变更」放在 `Refactor` 章节
- 「工具/构建/CI 变更」放在 `Chore` 章节

> 📌 **维护原则（给 Agent 看）**
> 1. 每次完成结构性变更（新增/拆分/迁移文件、依赖升级、构建脚本、API 行为变化）必须**在本文档对应版本下加一条**
> 2. 每条记录末尾用**括号标注平台**（见下方规则）
> 3. 旧版本不要修改，只在 [Unreleased] 或新版本下追加
> 4. 写完后 grep 验证段落结构没坏

## 平台/系统标注规则

每条记录末尾用括号标注涉及的平台，便于 Agent 快速判断是否需要考虑平台差异。

- **不标 = 默认全平台**（macOS / Windows / Linux/Ubuntu / Web）都行
- 常见标签：

  | 标签 | 含义 |
  | --- | --- |
  | `macOS` | 仅 macOS 客户端 |
  | `Windows` | 仅 Windows 客户端 |
  | `Linux/Ubuntu` | 仅 Linux/Ubuntu 客户端或服务器 |
  | `Server` | 仅服务端（部署在 Linux/Ubuntu 上的 Node 进程） |
  | `Web` | 仅浏览器端运行 |
  | `多端` | 涉及 2 个及以上平台 |

- 示例：
  - `（Server）` —— 仅服务端 Linux/Ubuntu
  - `（macOS）` —— 仅 macOS 客户端
  - `（多端）` —— 涉及 2 个及以上平台
  - `（macOS + Server）` —— 同时涉及两端

## [2.3.0] - 2026-07-08

> 发布：附件重命名闭环 + 代码规范强化。新增服务端 PATCH 重命名接口、附件循环导航与重命名行内编辑，同步修复空 catch 吞异常、CSS 硬编码色值、文件行数超标等多项规范违规。

### Changed
- 附件区点击跳转改为循环导航：单击附件跳到正文中的下一个引用位置，双击跳到上一个引用位置，避免多处重复引用时始终卡在第一处（Web）
- 附件右键菜单新增“重命名”，并将实际编辑收口到右侧附件行内完成，避免使用浏览器原生弹窗，同时保留轻量附件区布局（Web）
- 附件重命名输入改为只编辑主文件名，后缀在输入框右侧单独展示并默认沿用原扩展名；若原文件缺少后缀，则回落为 `.png`，避免误改附件扩展名（Web）
- 附件重命名前端调用改为 `PATCH` 优先、`POST /rename` 回退，并在接口缺失时改为侧栏状态提示而不是未处理 Promise 报错（Web）
- 附件重命名行继续收紧为固定宽度内联编辑态，右侧附件列表现在强制隐藏横向溢出，只允许纵向滚动，避免编辑态把整个附件区横向撑宽（Web）
- 图片插入后的后续书写逻辑改为"按图片块源地址定位，并在其正下方显式保证一个可编辑段落且把光标落入其中"，不再只依赖 Markdown 空行推断（Web）

### Added
- 服务端附件存储新增 `PATCH /api/storage/attachments/:id` 重命名能力，会同步更新附件元数据与磁盘文件名，供右侧附件区重命名闭环复用（Server）
- 服务端补充 `POST /api/storage/attachments/:id/rename` 兼容入口，降低不同代理/运行环境对 `PATCH` 的依赖（Server）

### Fixed
- 修复 `moveFileSafely`、`scheduleFileCleanup`、`deleteAttachment`、`importAttachmentsSnapshot` 中空 catch 吞异常的问题，补充 `console.error` 日志以便排查文件操作失败原因（Server）
- 替换附件区重命名表单的硬编码 CSS 背景色为设计令牌 `var(--color-accent-soft)`，并移除不匹配的左树 `library-inline-form` 样式复用，从根本上解决重命名表单横向错位问题（Web）

### Refactor
- 拆分 `local-attachment-store.js`（408→346 行）：将文件系统工具函数提取到独立文件 `local-attachment-store-utils.js`，消除行数超标与职责混杂问题（Server）
- 统一 `sidebar-controller.js` 返回对象的方法暴露风格，按来源分组本地方法、子控制器委托与展开（Web）

### Chore
- `event-bindings-controller.js` 中 click/dblclick 防抖阈值 220ms 添加设计注释，说明区分单击/双击的设计意图（Web）

### Tests
- `apps/web/test/events/aside-events.test.js` 新增附件单击/双击跳转方向断言；`navigation-context-menu-items.test.js`、`sidebar-renderers.test.js`、`knowledge-api.test.js` 补附件重命名入口与请求覆盖（Web）
- `apps/api/test/local-attachment-store.test.js`、`server-routes.test.js`、`app-factory.test.js` 新增附件重命名存储、路由和 handler 覆盖；当前 `npm run test:api` 仅剩一个既有基线失败：根路由文案仍断言 `Study Accelerator API`，而实际返回为 `知境·Knowra API`，与本次改动无关（Server）

## [2.2.0] - 2026-07-08

> 新增：附件第一阶段闭环，覆盖附件引用状态、正文跳转、重插入、从当前笔记移除与安全删除链路。

### Added
- 新增 `docs/附件第一阶段设计.md`，收敛笔记级附件闭环的第一阶段目标、删除策略、引用状态规则与后续全局附件库边界（Web）

### Changed
- 右侧附件区改为”列表项负责跳转、少量按钮负责高频操作、更多动作走右键菜单”的轻量交互：已引用附件点击可跳转到正文，附件行保留最小显式按钮，其余操作通过共享 context menu 承载（Web）
- 右侧附件第一阶段继续补齐闭环：未引用附件现在可通过右键菜单直接删除，形成”删正文图片 → 附件变未引用 → 在附件区清理”的安全两段式流程（Web）
- 附件引用状态判定统一收口到 `apps/web/lib/sidebar/attachments.js`：附件列表和右键菜单现在都优先基于当前草稿 Markdown 实时推导 `已引用 / 未引用`，不再各自散落做字符串判断（Web）
- 正文图片上传后的引用地址升级为显式附件引用 URL（`/content#attachment=<id>`），并在编辑器图片块 DOM 上同步挂出 `data-attachment-id`，让附件区跳转和引用判定优先按附件实体 ID 工作，而不是只靠原始 URL 文本比对（Web）
- 附件区右键菜单新增“插入到当前光标”动作：当前笔记中的附件现在可以直接从右侧列表重新插回正文，形成“上传/粘贴 → 删除正文引用 → 附件区保留 → 一键恢复插入”的第一阶段恢复闭环（Web）
- 附件区右键菜单新增“从当前笔记移除”动作：现在可以只删除当前笔记正文中的附件引用而保留附件文件与记录，执行后附件会立即回到 `未引用` 状态，形成安全的两段式清理/恢复链路（Web）

### Fixed
- 修复附件第一阶段的删除保护缺口：当前仍被正文引用的附件现在禁止直接删除，必须先移除正文引用再清理附件，避免用户在附件区误删仍在使用中的文件（Web）
- 修复右侧附件区底部右键菜单易被窗口底边遮挡的问题：共享 context menu 现在会在渲染后按视口尺寸自动回收位置，保证在辅助面板底部打开时也能完整显示（Web）

### Refactor
- 拆分附件命令到独立控制器 `sidebar/attachment-commands-controller.js`，`sidebar-controller.js` 保持只做面板状态与渲染编排（Web）

### Tests
- 新增 `apps/web/test/sidebar-attachments.test.js` 覆盖附件引用 ID 提取、URL 编码解码、显示层派生字段与正文附件引用删除 helper；新增 `apps/web/test/controllers/sidebar-controller.test.js` 覆盖“已引用附件禁止删除 / 未引用附件允许删除”两条保护路径（Web）
- `apps/web/test/knowledge-api.test.js` 现在断言附件上传返回 `referenceUrl`；`apps/web/test/v145-image-block-ui.test.js` 新增对 `data-attachment-id` 与附件感知引用 URL 的源模式校验（Web）
- 新增 `apps/web/test/controllers/editor-host-controller.test.js` 覆盖“附件重插入到当前光标”和“从当前笔记移除”的源模式断言；`apps/web/test/navigation-context-menu-items.test.js` 更新附件右键菜单动作顺序。web tests 116/116 通过（Web）
