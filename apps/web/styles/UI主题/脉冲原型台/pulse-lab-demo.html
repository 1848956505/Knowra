闭包为什么总让人“感觉神秘”

更新时间 2026-07-02 · 工作区 / JavaScript / 核心机制

很多人第一次理解闭包时，卡住的不是定义，而是感受。 你知道“函数记住了外层变量”，但真正困惑的是： 它到底记住了什么，又为什么在外层函数结束后还能继续工作？

1. 不是记住一个值，而是记住一段环境

闭包并不是把某个变量的“数值副本”偷偷带走了，而是让函数持续持有对原始词法环境的访问能力。 所以当环境中的值变化时，闭包访问到的也会是最新状态。

把闭包想成“带走了一把钥匙”，比“带走了一张快照”更接近真实。

2. 为什么这对 React、事件处理、异步回调都重要

只要一个函数被稍后调用，它几乎就离不开闭包。按钮点击、定时器回调、Promise 链、React Hook， 本质上都依赖函数在未来某个时刻重新拿到当前上下文里的东西。

这也是为什么 stale closure 问题会出现：不是闭包失效了，而是我们误判了“未来会读到哪一层环境”。

3. 一个更适合记忆的理解方式

把函数看成一个可移动的执行模块，把词法环境看成它的供电接口。 函数被带去别的地方执行时，只要这条供电链没有断，它就仍然能访问创建它时那一层作用域。
