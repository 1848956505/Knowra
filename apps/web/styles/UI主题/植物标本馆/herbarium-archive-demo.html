Specimen Note / Botanical Cognition

叶脉结构与记忆路径

把一篇笔记当作一张标本卡：主题是主脉，例证是侧脉，复习线索是通向下一次观察的编号。

VERIFIED
LOCAL
ARCHIVE

叶脉并不是装饰性的纹路，而是植物把养分输送到各个区域的结构。 在知识库中，笔记也需要这样的结构：一个概念不应该孤立地躺在页面里， 它需要能把定义、例子、反例、复习提示和相邻主题连接起来。

因此这张标本卡把 主题、分类、引用和复习线索 放在同一张纸面上。 左侧目录负责定位，中央纸面负责长文沉淀，右侧标本栏负责把这篇笔记归档到更大的知识系统里。

一条好的笔记路径应该像叶脉：既能从主干追踪到细节，也能从任意细节回到主题。

主脉：概念入口

主脉对应笔记标题与核心问题。它必须足够清晰，让未来的自己看到标题时就知道这张卡片解决什么问题。 对于学习型知识库，主脉不是一句漂亮文案，而是一个可以再次进入的认知入口。

OBS-07
标本馆式 UI 的重点不是把界面画满植物，而是让每个信息单元都像可检索、可归档、可复访的馆藏资料。

侧脉：例证与引用

侧脉连接正文中的例证、引用、外部资料和相关笔记。右侧栏不抢夺阅读注意力， 但保持可扫描的馆藏标签形态，让读者知道这张标本卡属于哪个分类，能通向哪些相邻卡片。
