闭包为什么让“过去”和“现在”看起来像连在一起”

闭包最容易让人误会的地方，是它看起来像把过去留在了现在。一个函数离开原来的位置后，竟然还能继续访问外层变量，这让很多人直觉上觉得它“记住了旧值”。

但更准确的理解是：闭包延续的是访问环境的通路，而不是冻结了一份值的快照。

Grinding Insight

为什么“钥匙”比“照片”更适合类比闭包

如果把闭包当成照片，就会默认它只保留一个固定时刻的内容；但如果把它看成钥匙，就更容易理解为什么函数到了未来仍然能回去取材料。

闭包保存的不是一段旧记忆，而是一条仍然可达的访问路径。

为什么异步回调和 React 都绕不开它

只要函数不是立刻执行，而是留给未来某个时刻使用，它就天然带上了闭包属性。这也是为什么 stale closure 在 React 里反复出现，本质上是对“当前函数到底引用哪一层环境”的判断出错。
