闭包 · 词法环境与捕获

闭包是 JavaScript 中最容易被误用的概念之一。它不是"函数返回函数"，而是函数与其词法环境的绑定。

基本形态

function makeCounter() {
  let count = 0
  return function() {
    return ++count
  }
}

const counter = makeCounter()
counter() // 1
counter() // 2

关键点

  • 捕获的是变量本身，不是变量在某一时刻的值
  • 多个闭包共享同一份词法环境
  • 内存释放取决于引用关系，不取决于作用域结束

常见误用

在循环中创建闭包时，var 的提升会让所有闭包共享同一个变量。

设计 token 体系笔记

设计 token 是设计系统中最基础的语义化变量。它把"颜色"这种视觉值与"按钮主色"这种角色分离开来，让主题切换只动 token、不动组件。

三层结构

  • 原始层：色板、字阶、间距阶梯等原始值
  • 语义层：主品牌色、强调色、危险色等角色
  • 组件层：按钮底色、卡片投影、tab 激活条等

原始层示例

--color-mint-400: #2DD4BF;
--color-aurora-500: #8B5CF6;
--gradient-aurora: linear-gradient(135deg, #2DD4BF, #8B5CF6);

语义层映射

--color-accent: var(--color-mint-400);
--color-accent-soft: rgba(45, 212, 191, 0.12);
--color-glow: 0 0 0 4px rgba(45, 212, 191, 0.16);

使用注意

永远不要在组件中写死色值（#xxxxxx）。所有视觉值都从 token 走。

更多参考：W3C Design Tokens spec · Style Dictionary

任务清单

  • 定义原始层色板
  • 定义语义层映射
  • 组件层接入
  • 主题切换验证
